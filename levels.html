Documentation Levels

For each architectural lens, information is categorized into three consistent levels of detail. This approach ensures that documentation caters to diverse audiences, from those seeking a high-level overview to those requiring deep technical specifications for implementation or audit.

Level 1 — Introduction

Purpose:

Build basic mental model.

Audience:

Newcomers, executives, adjacent teams.

Characteristics:

  • Narrative over detail
  • Visual first
  • No prerequisite knowledge assumed
  • Explains why this lens matters

Level 2 — Orientation

Purpose:

Enable navigation and targeted lookup.

Audience:

Engineers, architects, reviewers.

Characteristics:

  • Structured overview
  • Key artefact references
  • Terminology defined
  • Shows “where to find more”
  • This level answers: “Where does X live?”

Level 3 — Elaboration

Purpose:

Enable implementation, analysis, or change.

Audience:

Implementers, auditors, deep reviewers.

Characteristics:

  • Detailed diagrams, specs, contracts
  • Links to living artefacts (e.g., OpenAPI, Terraform, ADRs)
  • Trade-offs documented
  • Edge cases covered
  • This level answers: “How exactly does this work?”
