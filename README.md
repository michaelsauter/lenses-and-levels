# Lenses & Levels Architecture Documentation Model

A novel approach to software architecture documentation, combining different "lenses" of architectural concerns with consistent "levels" of detail for each lens.

## Overview

The "Lenses & Levels" framework organizes documentation around two key axes:

- **Lenses:** Each lens represents a distinct architectural concern (System, Capability, Data, Security, etc.).
- **Levels:** For each lens, information is presented across three consistent levels of detail: **Introduction**, **Orientation**, and **Elaboration**.

## Key Features

- **Targeted Information:** Stakeholders can quickly find relevant detail.
- **Consistent Structure:** Predictable navigation across all lenses.
- **Scalability:** Accommodates high-level overviews to deep technical specs.
- **Modern UI:** Built with clean HTML and CSS for readability.
