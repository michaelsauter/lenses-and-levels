UX & Interaction Lens

Description: How the user perceives and interacts with the system. Covers user journeys, design systems, frontend architecture patterns, and accessibility.

General Level Guidance

Level 1 — Introduction

  • Who are the target user personas and what is their primary interaction model?
  • What is the 'Visual Language' or brand identity of the interface?
  • What are the core user journeys that define the product experience?

Common Fulfillment: A Design Vision statement and high-level User Journey maps.

Level 2 — Orientation

  • What design system or component library is used (e.g. Material UI, Tailwind)?
  • How is the frontend structured (e.g. Micro-frontends, SPA, SSR)?
  • Where are the UI assets and design specs (e.g. Figma, Storybook) located?

Common Fulfillment: Link to Figma files and a Storybook instance showing the component library.

Level 3 — Elaboration

  • How is frontend state managed (e.g. Redux, XState, TanStack Query)?
  • What is the specific accessibility (WCAG) compliance strategy?
  • How are complex interactions or animations orchestrated?

Common Fulfillment: Detailed interaction diagrams, accessibility audit reports, and state transition charts.

Example: Completely over-engineered Todo-App

Level 1 — Introduction

The UI is designed for the 'High-Velocity Professional' who values keyboard-first interactions and instant feedback. The visual style is 'Hyper-Minimalist' to reduce cognitive load during task management.

Level 2 — Orientation

We use a custom design system built on Tailwind CSS and Radix UI primitives. All UI components are documented in our internal Storybook. The architecture is a React SPA using Next.js (App Router) for optimized loading and SEO of public lists.

Level 3 — Elaboration

Frontend State Strategy

We use TanStack Query for server-state synchronization with optimistic UI updates. When a user marks a task as done, the UI reflects the change immediately before the API response is received, reverting only on failure.

Accessibility (A11y)

The app is WCAG 2.1 Level AA compliant. We use aria-live regions for real-time sync notifications and ensure a logical tab-order for all list management functions.
